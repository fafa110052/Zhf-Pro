## 一、优先修复与完善（v1.0.1）

上线前应完成的高优先级项：

### 1.1 微信小程序正式注册

- [ ] 在 [mp.weixin.qq.com](https://mp.weixin.qq.com) 注册小程序账号，获取正式 AppID 和 Secret
- [ ] 将 AppID 填入 `server/.env` → `WECHAT_APPID` 和 `WECHAT_SECRET`
- [ ] 将 AppID 填入 `miniprogram/project.config.json` → `appid`
- [ ] 设置 `WECHAT_DEV_MODE=false`
- [ ] 提审微信小程序（首次提审约 1-7 个工作日）

### 1.2 安全加固

- [ ] 生产环境 JWT_SECRET 更换为随机 64 位十六进制字符串
- [ ] 管理员默认密码 admin123 首次登录强制修改
- [ ] API 请求频率限制（express-rate-limit）
- [ ] 上传文件 MIME 类型严格白名单校验
- [ ] SQL 注入审计（Knex 参数化查询已做基础防护）
- [ ] CSP 头配置（Helmet 中间件）

### 1.3 数据完整性

- [ ] 作品删除时关联图片的物理文件清理
- [ ] 图片库 orphan 图片（无引用）的定期清理任务
- [ ] 分类删除时的级联策略细化（目前硬阻止 409）

## 三、性能优化（v1.2）

### 3.1 后端

- [ ] **图片处理优化**：sharp 调整为异步 pipeline，大量上传时不阻塞事件循环
- [ ] **分页游标**：大数据量下 keyset 游标分页替代 offset/limit
- [ ] **慢查询优化**：添加复合索引、定期 ANALYZE
- [ ] **响应压缩**：`compression` 中间件 gzip
- [ ] **静态资源缓存**：`Cache-Control` 头配置

### 3.2 管理后台

- [ ] 路由懒加载（React.lazy + Suspense）
- [ ] 图片列表虚拟滚动（react-window）
- [ ] Dashboard 数据 SWR/stale-while-revalidate

### 3.3 小程序

- [ ] 首页数据预加载（`onReachBottom` 提前一页）
- [ ] 图片懒加载优化（当前已加 `lazy-load`，可调阈值）
- [ ] 模板消息订阅（审核结果通知）
