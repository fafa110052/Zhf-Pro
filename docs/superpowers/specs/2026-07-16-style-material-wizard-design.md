# 风格选材向导 — 设计规格说明书

> 状态：已确认 | 日期：2026-07-16 | 版本：v1.0

## 1. 概述

在现有「住好房」小程序中新增「风格选材」tab，与旧「在线选材」tab 并存。客户选定装修风格后，通过六步向导逐品类完成全屋材料配置，最后生成带优惠价的选材清单提交公司。

**与旧「在线选材」的关系**：两套选材完全独立，数据表、API、后台管理均不共享。旧选材完整保留不动。

## 2. 用户旅程

```
tab「风格选材」（新增）
  → ①选六大装修风格
  → ②六步选材向导（逐页，底部"下一步"）
  → ③选材总结 + 填业主信息 + 原价划线→优惠价
  → ④提交成功
```

- 中途退出自动存草稿，下次进入从断点续选
- 顶部常驻六步步步骤条
- 未登录可浏览，保存草稿/提交需登录

## 3. 六页选材设计

### 3.1 瓷砖选材（第1/6步）

三个板块从上到下：

| 板块 | 内容 | 布局 | 文字字段 |
|---|---|---|---|
| A | 客厅、餐厅、厨房、房间、阳台地板砖 | 上图下文 | logo + **品牌** + 型号 + 规格 |
| B | 厨房、卫生间墙砖 | 上图下文（与A微调） | 同A |
| C | 卫生间地板砖 | 左图右文，一行2个 | 同A |

每个板块单选一项。全选完后"下一步"亮起。

### 3.2 室内木门（第2/6步）

门系列卡片（布拉格/罗塞蒂/麦加等，带序号+醒目标题），每个系列内展开可选颜色（色块图+颜色名，单选）。先选系列再选颜色，组成一个完整选择。

### 3.3 卫浴选材（第3/6步）

七个子板块：

1. 浴室柜组合（镜柜+主柜+台面，上图下文，一行2个）
2. 卫生间门（颜色选择 + 锁向：左锁右内开/右锁左内开）
3. 马桶（图文，型号+规格+排水方式）
4. 蹲厕（图文，型号+规格+前出水墙距）
5. 水箱（图文，型号+水箱规格）
6. 花洒（图文，型号）
7. 水龙头（图文，型号）

每个子板块单选。卫生间门有两层独立选择（颜色+锁向）。

### 3.4 沙发选材（第4/6步）

图文卡片，单选。文字：型号+材质+面质+规格/Size。若后台标记"有贵妃"，选中后展开左贵妃/右贵妃选择。

### 3.5 家具选材（第5/6步）

五个子类目各单选：

| 类目 | 文字字段 |
|---|---|
| 床 | 标题、规格(1.8M/1.5M)、材质、面料 |
| 餐桌餐椅 | 标题、餐椅(换行展示)、规格/Size |
| 电视柜 | 标题、规格/Size |
| 茶几 | 标题、规格(可多行组合) |
| 床头柜 | 标题、规格/Size |

### 3.6 装饰灯具（第6/6步）

9个套餐卡片，单选。每套餐含5件固定搭配：主厅灯×1 + 餐厅灯×1 + 卧室灯×3。套餐内不可单独替换。每件灯具展示：尺寸/功率/材质/颜色/光源/控制方式/照射面积/全国统一零售价(红色加粗)。

## 4. 缺失环节补全

以下环节老板未描述但产品闭环必需：

- **风格选择页**（向导前）：六大风格大图卡片入口
- **总结提交页**（向导后）：分页展示已选项+价格汇总+业主信息+提交
- **草稿机制**：自动存/断点续选/我的选材中查看
- **进度条**：顶部分步步骤条 + 页内子进度

## 5. 关键盲点与决策

| # | 严重度 | 问题 | 决策 |
|---|---|---|---|
| 1 | 🔴 | 内容生产瓶颈 | 先录满1个风格走通，其余复制调整；开发期用占位图 |
| 2 | 🔴 | 产品属性多样化 | JSON弹性字段，品类定义属性模板，后台动态渲染 |
| 3 | 🟡 | 灯具零售价 vs 原价/优惠价 | 套餐卡标"建议零售价"(红)，总结页标原价→优惠价 |
| 4 | 🟡 | 门颜色跨系列隔离 | 颜色挂系列下，不建全局颜色表 |
| 5 | 🟡 | 沙发贵妃 | materials表has_chaise布尔字段，前端动态展开 |
| 6 | 🟡 | 茶几组合(圆+方) | 当作一个商品，规格描述组合内容 |
| 7 | 🟡 | 卫生间门颜色+锁向 | 卡内两个独立选择区 |
| 8 | 🟢 | 蹲厕/水箱关联 | 一期独立选 |
| 9 | 🟢 | 床1.8M/1.5M | 同一商品，规格描述两种尺寸 |
| 10 | 🟢 | 允许跳过类目 | "此项不选"按钮，总结页标注 |

## 6. 数据模型

### 核心表

**styles** — 装修风格：name, cover_image, description, sort_order, enabled

**categories** — 品类页（6条预设）：name, page_number(1-6), sort_order

**subcategories** — 子品类（页面内板块）：category_id, name, sort_order, layout_type, columns

**materials** — 材料/产品：subcategory_id, name, model, brand, brand_logo, image_url, original_price, discount_price, specs, attributes(JSON), has_chaise

**material_styles** — 材料-风格关联(多对多)

**door_series** + **door_colors** — 门系列与颜色（颜色挂系列下）

**lighting_packages** + **lighting_package_items** — 灯具套餐与5件明细

**selection_drafts** — 草稿：user_id, style_id, current_step, data(JSON)

**selection_orders** — 选材单（新表）：order_no, user_id, style_id, 业主信息, original_total, discount_total, items(JSON快照), status, 预留designer_id/supervisor_id

### 关键决策
- 弹性属性用JSON（每个子品类配置属性模板）
- 门系列、灯具套餐独立建模
- 选材单价格JSON快照（锁定提交时刻价格）
- 不复用旧material_orders表

## 7. 后台管理

新增7个页面：风格管理、品类管理、材料管理、门系列管理、灯具套餐管理、选材单管理。遵循现有admin-ui-design-system规范。

## 8. UI/UX 规范

- **配色**：暖白奢华 — 米白底#FAF8F5 / 炭黑字#1A1A1A / 香槟金#C9A96E
- **选中态**：金边+微缩放+对勾动画
- **图片**：点击lightbox全屏，左右滑切换同类目选项，放大态可选中
- **步骤过渡**：右进左出横滑
- **标题**：借鉴PDF编号+中英文版式

### 8.1 图片放大查看（Lightbox）

所有六页中的材料/产品图片均支持点击放大查看，这是提升"商品感"和选材爽感的核心交互：

- **触发**：点击材料卡片中的产品图片 → 全屏 lightbox 打开
- **手势操作**：
  - 左右滑动 → 切换到同子品类下的上一张/下一张产品图（无需退回列表）
  - 双指捏合 → 自由缩放查看纹理细节
  - 下滑 → 关闭 lightbox 回到列表
- **放大态内选中**：可在 lightbox 内直接点"选中"按钮，选中后缩略图回退继续浏览同类目其他选项——实现"边看大图边选"的沉浸体验
- **顶部信息栏**：显示当前产品名称 + 类目内进度（如"3/8 地板砖"）
- **小程序实现**：用 `wx.previewImage` 的 `current` + `urls` 传同类目所有产品图列表，原生支持左右滑切换；或自建 lightbox 组件实现"放大态可选中"（推荐自建，控制力更强）
- 实施时调用 ui-ux-pro-max 出详细视觉稿

## 9. 价格体系

- original_price：商品卡片+总结页划线展示
- discount_price：商品卡片+总结页醒目展示
- 灯具retail_price：明细锚定价，套餐original/discount控制最终价格
- 总结页：Σ原价(划线) → Σ优惠价(大字)
- 不按面积/数量：一期全是一口价

## 10. 分期

**一期（MVP）**：后端迁移+API → 后台7页面 → 小程序8页面(风格选择+六步向导+总结提交+我的选材) → 联调

**二期**：工程派单、面积精算、多方案对比、清单导出

**三期**：客户自助修改、变更记录、选材数据分析
